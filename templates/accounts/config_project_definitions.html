What “Operating Profile” means

A project’s Operating Profile is the project-scoped Level 4 configuration record. It expresses how work should run by default inside this project: work type, mode, evaluation posture, checkpointing posture, and off-ramps.

What it does

  • Provides defaults when a user starts work in the project.
  • Gives a stable, auditable “starting posture” for project work.
  • Enables consistent behaviour across users and sessions for that project.

What it does not do

  • It does not override Level 3 governance or sovereignty controls.
  • It does not grant authority (authority is via roles, not config).
  • It does not make chat history canonical.

Resolution order (effective configuration)

  1. User defaults (Level 1)
  2. Project defaults (Level 4 Operating Profile)
  3. Session overrides (ephemeral)
