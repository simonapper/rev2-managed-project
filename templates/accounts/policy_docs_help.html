Use this later when you are ready to load policy docs.

Expected fields
  • Title: short name for the policy.
  • Source ref: optional provenance, version, or URL.
  • Body text: plain text content used for keyword retrieval.
Recommended format
  • Use clear section headings in plain text.
  • Keep dates and version info in source ref.
  • Use tags in boundary constraints, for example UK_LAW, COMPANY_POLICY.
LLM pre-processing

You can ask the LLM to convert raw policy files into clean text.

  • Remove tables/images that do not matter for retrieval.
  • Keep obligations, thresholds, exceptions, and deadlines.
  • Paste the final cleaned text into Body text.
